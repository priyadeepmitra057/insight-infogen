# Subsystem: 09a Training & Data Generation

## Files in Scope
- `training_data_generator.py` (lines 142-200) (CORE_LOGIC)
- `train_and_save_models.py` (ENTRYPOINT)

## 1. Responsibility
- **Owns**:
    1. Generating synthetic, mathematically consistent transaction datasets (`training_data_generator.py` (lines 142-200)).
    2. Acting as the executable script to train the offline `insight_ranker.pkl` model and save it alongside its SHA-256 checksum (`train_and_save_models.py`).
- **Does NOT own**: The definition of the Insight Model pipeline itself (that lives in `insight_model.py`).

## 2. Public Interface

### `generate_insight_dataset(n_samples=5000, random_state=42) → pd.DataFrame`
- **File**: `training_data_generator.py` (lines 142-200)
- **Preconditions**: None.
- **Postconditions**: Returns a dataframe containing all required numeric and categorical features, plus a `TARGET_IS_INSIGHT` label.
- **Failure modes**: None.

### `train_and_save() → None`
- **File**: `train_and_save_models.py`
- **Preconditions**: None.
- **Postconditions**: Writes `models/insight_ranker.pkl` and `models/insight_ranker.pkl.sha256` to disk.
- **Failure modes**: Disk I/O errors.

## 3. Internal Design
- **Key algorithms**:
    - *Synthetic Data Constraints*: The generator explicitly injects edge cases (e.g. `is_anomaly = True` combined with `category_confidence < 0.2`) to ensure the downstream ML model learns the boundaries of the business rules. It uses probability distributions (`np.random.normal`, `np.random.beta`) to simulate realistic financial data.
    - *Model Fitting*: Fits a `LGBMClassifier` on the synthetic data, serializes it, and hashes the byte string for security.
- **State model**: Procedural.

## 4. Issues & Risks
- **RISK**: Synthetic Data Drift.
    - *What it does*: The Insight Ranker is trained on purely synthetic data generated by `training_data_generator.py` (lines 142-200).
    - *Why it is a problem*: If real-world transaction patterns drift away from the hardcoded distributions in the generator, the insight model will become increasingly useless, and there is no feedback loop to fix it since it's not trained online.
    - *Suggested fix*: The pipeline should support fine-tuning or replacing the offline model with a model trained on real, anonymized production data.
